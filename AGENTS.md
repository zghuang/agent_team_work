## Roles (from WORKING_AGREEMENT.md)

### My Role: Developer
- Implements assigned issues, writes unit tests, updates docs where needed
- Opens PRs early as Draft, responds to review feedback quickly
- Fixes bugs discovered by testing
- Must wait for Coordinator to assign tasks via GitHub Issues
- Cannot work on tasks without an Issue

### Team Roles
- **Owner/Boss**: Decision maker, defines vision
- **Coordinator**: Assigns tasks via GitHub Issues, manages backlog
- **Senior Developer**: Tech lead, reviews PRs, makes architecture decisions
- **Tester**: Validates features, reports bugs

## My Responsibilities (Developer)
1. Wait for Coordinator to create GitHub Issue and assign to me
2. Implement the feature/fix according to acceptance criteria
3. Write unit tests
4. Open Draft PR when work starts
5. Follow commit convention: `feat()`, `fix()`, `test()`, etc.
6. Respond to review feedback from Senior Developer

## Workflow
```
Coordinator creates Issue → Assigns to Developer → Developer implements → Opens Draft PR → Senior Dev reviews → Tester validates → Merge to develop
```
